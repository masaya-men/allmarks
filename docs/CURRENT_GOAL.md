# 次セッションのゴール — リリース（公開）までの計画を立てて進める

## 進め方（最速プロトコル）

- **1 機能ずつ、フレッシュなセッションで**進める。設計は最小往復で最速、実装→検証→デプロイまで自走。
- 見た目変更は claim 前に Playwright で実測（ユーザーが「確認は自分でやる」と言ったら省略可）。応答は短く。戻せる修正は逐一承認を取らず自走。デプロイは `--branch=master`。
- **機微（収益化・価格・ローンチ戦略・競合）は必ず `docs/private/IDEAS.md`**。tracked ファイルには書かない。

## 直近の完了（s176）＝ SHARE 自動画像化 出荷＋「本物のボード枠込み」化（本番OK）

- **SHARE = 「①選ぶ ②並べる ③作る」の1ボタン**。手動スクショ撤廃。本物のアレンジ画面を dom-to-image で自動撮影（同一オリジン画像 proxy `functions/api/img.ts` でクロスオリジン汚染回避）→ 1200×630 → 既存 R2／`/og/<id>.jpg` → LINK READY。
