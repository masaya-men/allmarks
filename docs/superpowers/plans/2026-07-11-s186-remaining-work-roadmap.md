## 2. ローンチブロッカー（A群）

- **(N-56) ★★スマホで共有画像が作成されない** — 実機のみで再現。設計上、撮影失敗でもリンクは作られるため「静かに壊れて成功に見える」。→ 詳細計画書あり（症状の切り分け＋可視化＋段階フォールバック＋条件分岐の修正タスク）。
- **(N-58) ★スマホでもコラージュさせる** — s185 spec §2.1「並べる段は出さない」をユーザーが実機体験で撤回。「簡素でもコラージュしたい。表現の場なのでスマホでもきちんと表現させたい」。→ 詳細計画書あり（段階1=編集で止める＋CAPTURE、段階2=ピンチズーム。N-55 も同時に消える）。
- **(N-57) スマホ盤面に背景タイトルが出ない** — `BoardBackgroundTypography` の `!isMobile` ゲート。出した後「共有画像にもタイトルを載せるか」の判断が付随。→ 詳細計画書あり。
- **(N-59) スマホの列数・余白を簡易調整** — 離散選択（列 2/3/4・余白 小/中/大）。`MOBILE_LAYOUT` は現在ベタ書き定数。IDB `board-config` の JSON 塊に載せれば DB 版数上げ不要。→ 詳細計画書あり。

## 3. ローンチ前に強く推奨（B群）

## 7. モデル割り当ての原則（s185 実測に基づく）

- **Haiku**: 計画書に実コード・実テスト・実コマンドが全部埋まっている純関数/CSS/定数タスク。判断ゼロで写経できる粒度のもの。
- **Sonnet**: 配線タスク（BoardRoot 等の大ファイルへの挿し込み）、Playwright 実測、翻訳レビュー（束C は Haiku 不可）。
- **Opus/Fable**: 設計判断・診断の読み解き・全ブランチレビュー・brainstorm。
- 各詳細計画書のタスク見出しに推奨モデルを明記してある。**迷ったら1つ上のモデル**。

## 8. 恒久ルール（全セッション共通・計画書より優先）

- **ボードの見た目を変えたら受け取り画面（`/s/<id>`）も必ず確認**（s183/s184 の事故の再発防止）。
- **撮影まわりは実機でしか検証できない**（Playwright 5/5 緑でも実機で壊れた実績あり）。実機確認はユーザーに依頼する。
- `CardsLayer` に `isMobile` を渡さない（`×`・タグピルが消える）。スクロール解放は `lockCardScroll`。
- カードのタップ選択は `setPointerCapture` 使用＝Playwright で駆動不可。`SELECT ALL` 経由で確認。
- **`fit:'cover'` を `'contain'` に戻さない**（`tests/e2e/mobile-share.spec.ts` の黒帯検出テストが守る）。
- デスクトップ（>640px）の描画は 1px も変えない。新 UI は `useIsMobile()` / `@media (max-width:640px)` ゲートの内側。
- 検証してから claim: `rtk tsc` / `rtk vitest run` / `pnpm build`（`rtk next build` 不可）→ 必要な Playwright 実測 → デプロイ（`npx wrangler pages deploy out/ --project-name=allmarks --branch=master --commit-dirty=true`）。
- `--no-verify` 絶対禁止。機微（戦略・価格・実メアド）は tracked に書かない。
