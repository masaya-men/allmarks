# Design (direction) — フラット化：白フラット default テーマ＋テーマ境界の確定 (N-26/32/33/35)

日付: 2026-07-05 / セッション 162
種別: **方向性（親）spec**。大物のため下記の順にサブへ分解し、各サブが独自の詳細 spec→plan→実装を持つ。

## 背景・動機
LP は白/エディトリアルのフラットデザイン。初回来訪者の盤面がそれと呼応するよう、**フラットな白テーマを新デフォルト**にしたい。同時に、これからテーマを増やすにあたり「各テーマが各メニューを個別に飾る」と**必ず“抜け”（未対応で崩れる箇所）が出る**。そこで**テーマが乗る範囲を盤面だけに絞る**。現在の見た目（気に入っている音波モチーフ）は**丸ごと温存**する。

## 確定した決定（session 162 ユーザー合意）
1. **新デフォルト＝白フラット・エディトリアルテーマ**（LP と呼応）。`DEFAULT_THEME_ID` をこれに差し替え。
2. **現在の暗い体験＝「音波（Sound Wave）」テーマとして温存**（波形メーター＋wave/グリッチ＝カードモーション＋暗い格子）。盤面アイデンティティは 1 バイトも変えない（現 `dotted-notebook` を土台に温存）。
3. **テーマが乗る範囲＝盤面のみに限定**。新テーマが定義するのは実質**5項目だけ**：
   - 背景/パターン（`backgroundClassName`＋customization の pattern 系）
   - カードの見た目
   - スクロールメーター（`scrollMeterVariant`）
   - カードのモーション（`motion.entry/text/shutdown`）
   - `colorScheme`（明暗）
4. **メニュー類＝全テーマ共通の“中立 chrome”に固定**（テーマは一切乗らない＝抜けが構造的に起きない）。**例外なし**：paper-atelier のメニューも中立へ戻す。
5. **出方の統一**：
   - **大パネル（TUNE / SETTINGS / SHARE / テーマ選択）＝右からのドロワーに統一**（従来の“右スライド”を唯一の標準に昇格）。
   - **小さな文脈ポップ（フィルタ・タグ追加）＝ポップのまま**（ドロワー化しない）。ただし chrome は同じ中立語彙。
   - フラットでは派手な個別メニュー演出を退役し、**中立 chrome の落ち着いた1つの右スライド**に一本化。
6. **カスタマイズ追加**：カード角丸 **ON/OFF**（`--card-radius` に流す新フィールド）。加えて N-35 のつまみ（格子線の太さ・ドット径・タイトル font/サイズ）を customization 層へ。既存の edge/board/pattern/title 色・pattern 種別/密度は流用。
7. **N-33 タグ表記**：現在アプリ全体 `text-transform:lowercase`。フラットの視覚言語の一部として決める（大文字許可 or 小文字維持）。**サブ④で確定**（勝手に剥がさない）。

## byte-identical 不変条件の扱い（重要）
これまでの「default 盤面 byte-identical」は**本プロジェクトで意図的に更新される**：
- **default が白フラットに変わる**＝初回の見た目が変わるのは意図した目的。
- **暗い盤面の保全パス＝音波テーマ**（盤面は byte-identical に温存）。
- ただし**メニューは全テーマで中立化**するため、音波テーマでも「盤面は従来どおり／メニューは新・中立」になる（＝メニューは byte-identical でなくなる。これは合意済みの意図的変更）。

## 実装機構（確認済み・素直に載る）
- テーマ登録＝[theme-registry.ts](../../../lib/board/theme-registry.ts)（`THEME_REGISTRY` に1つ足し `DEFAULT_THEME_ID` を差し替え）。
- customization＝[theme-customization.ts](../../../lib/board/theme-customization.ts)（`ResolvedThemeCustomization` に角丸・線幅・ドット径・タイトル font/サイズを追加。per-theme デフォルトで byte-identical を担保）。
- 明るい盤面で chrome 文字を黒インクへ反転する `isLightColor` は既存＝白フラットでも文字が消えない。
- テーマ永続＝`localStorage['allmarks-theme-id']`／初回は `DEFAULT_THEME_ID`。

## 分解と順序（各サブが独自 spec→plan→実装）
- **① テーマ境界の確定＋メニュー中立化＋右スライド統一**（基盤リファクタ）。まずここ。テーマ可変範囲を盤面5項目に閉じ、全メニューを中立 chrome＋大パネルを右ドロワーに。paper のメニュー中立化を含む。
- **② 白フラット default テーマ**（新規追加＋default 差し替え。パレット・メーター・モーション“無し”を詳細化）。
- **③ カスタマイズ拡張**（角丸 ON/OFF＋N-35 つまみ）。
- **④ 音波テーマの命名＋N-33 タグ表記の確定**。

## 非対象（今回のフラット化から切り離し）
- **N-27（左右マージンのスナップ）**＝レイアウト/操作の別件。別途。

## 各サブで詰める未確定（本 spec では方向のみ・詳細はサブ spec）
- ② 白フラットの具体パレット（LP トークンとの対応）、メーターの静かな見せ方、モーション“無し”の正確な定義（登場/テキスト/退場）。実際の見た目はモックで確認してから確定。
- ① 中立 chrome の視覚言語（右ドロワーの余白/罫/タイポ）、どの小ポップを据え置くかの最終線引き。
- ③ N-35 つまみの最終セットと露出範囲（default は静かに・でも“表現の摘み”は残す）。
