# SHARE アレンジ「盤面を端まで埋める」詰め方（justified rows）設計

- 日付: 2026-07-06（セッション168）
- 種別: 設計書（spec）
- 対象: SHARE のアレンジ段でカードを自動配置する純関数 `fitSelectionToScreen`
- 前段: [2026-07-06-share-arrange-fit-to-screen-design.md](2026-07-06-share-arrange-fit-to-screen-design.md)（s167・skyline＋一律縮小）を**詰め方だけ差し替える**。安全領域・座標焼き込み・移動/リサイズ/回転・装飾追従は前段のまま流用。

---

## 1. 背景と問題（確定した根本原因）

s167 末、ユーザー実機（赤マーク付きスクショ）で未解決のフィードバック：

> SHARE アレンジで100枚を全選択すると、カードが**左上に固まり、右の縦帯・下の大きな帯が空く**（L字の余白）。この空白が最小になるよう**盤面全体に詰めてほしい**。

これは「カード同士の隙間（gap）」の話ではない（gap は s167 で16pxに縮めて解決済み）。**盤面という長方形を充填しきれていない**のが問題。

根本原因：現行 `fitSelectionToScreen` は「石垣状パック（skyline masonry）＋収まる最大倍率の一律縮小＋中央寄せ」。
- 石垣は**右端がギザギザ**（次のカードが入らない所に隙間）・**下端も列で不揃い** → 矩形を埋めない。
- 中身の縦横比（contentW:contentH）は盤面矩形の縦横比と一般に一致しない → 中央寄せで**必ず片側が余る**。ウルトラワイドで最悪（実測で幅~89%・高さ~77% ＝ 右＋下のL字）。

## 2. 設計判断：「3つは同時に満たせない」トレードオフ

コラージュに欲しい性質は3つあり、**同時には2つまで**しか満たせない：

| 方式 | ①矩形を埋める | ②縦横比を保つ | ③相対サイズを保つ | 結果 |
|------|:--:|:--:|:--:|------|
| 現行（石垣＋一律縮小） | ✗ | ✓ | ✓ | **L字の余白**（今の不満） |
| **justified rows（採用）** | ✓ | ✓ | △ | 端まで充填／サイズは行ごとに揃う |
| モザイク（面積分割・treemap） | ✓ | ✗ | ✓ | 端まで埋まるが**カードを切り抜く**（却下） |

- **②は絶対制約**：カードは本物の board カード面（画像/動画＋タイトル）を描画する（`CollageCanvas` → `pickCard`）。縦横比を崩すとタイトル等が歪む。
- ユーザー確定：**③相対サイズは完全に無視してよい**（「盤面で拡大されていたかどうかはすべて無視」）。
- よって **①を取り③を捨てる = justified rows** を採用。

**トレードオフの緩和**：自動配置は「良い初期状態」であって最終形ではない。配置後も**掴んで移動・拡大・リサイズ・回転は従来どおり**使える（前段の仕組みを維持）。端まで埋める初期状態から、主役カードだけ手で引っ張って作れる。

## 3. 採用レイアウト：justified rows（写真ギャラリー方式）

Google フォト / Flickr の並べ方。カードを document 順に行へ流し込み、**各行を rect 幅いっぱいにピッタリ引き伸ばす**（同じ行のカードは高さが揃い、幅は縦横比なりに可変）。行を縦に積み、**行数を選んで rect 高さも埋める**。縦横比は保ったまま右も下も端まで充填する。

### 3.1 使うカード情報は「縦横比だけ」

各カードから使うのは **アスペクト比 `a = width / height`** のみ（`itemSkylineHeight` により `height = width / aspectRatio` なので `a = aspectRatio`）。絶対幅（`customWidths` / `cardWidthPx`）は使わない ＝ ③相対サイズ無視がそのまま実現。

### 3.2 盤面の設定値を2つ流用（ユーザー合意）

`lib/board/constants.ts` の `BOARD_SLIDERS` から：

- **サイズ上限** `MAX_CARD_WIDTH = CARD_WIDTH_DEFAULT_PX`（= 267.84px）。カードのレンダリング幅はこれを超えない。
- **隙間比** `GAP_RATIO = CARD_GAP_DEFAULT_PX / CARD_WIDTH_DEFAULT_PX`（= 97.21 / 267.84 ≈ 0.363）。隙間は**カードの大きさに比例**（盤面と同じ比率）。カードが上限サイズのとき隙間 ≈ 97px（盤面そのもの）、縮むと隙間も同率で縮む。

### 3.3 1行の幾何（縦横比＋隙間比から行高が閉形で決まる）

行にカードが `n` 枚、アスペクト和 `S = Σ aᵢ`、隙間は各カード高さに比例（`gap = GAP_RATIO × h`）。行を rect 幅ちょうどに満たす行高 `h` は：

```
S·h + (n-1)·(GAP_RATIO·h) = rect.width
⇒ h = rect.width / ( S + (n-1)·GAP_RATIO )
```

- 各カード：幅 `= aᵢ · h`、高さ `= h`。
- 行内・行間の隙間 `= GAP_RATIO · h`。

### 3.4 サイズ上限（blow-up 防止＝少数カードが board-ish に収まる仕組み）

行を幅いっぱいに満たすと、少数カード（特に横長）は board 既定より大きく育つ。これを防ぐため**その行の最大幅カードが `MAX_CARD_WIDTH` を超えないよう行高を頭打ち**する：

```
h_row = min( h_fill,  MAX_CARD_WIDTH / maxAspect_in_row )
```

- 上限が効かない（多数カードで `h_fill` が十分小さい）とき → 行は幅いっぱいに充填。
- 上限が効く（少数カード）とき → カードは board 既定幅で止まり、その行は幅を満たさない → **水平中央寄せ**（無理に巨大化しない）。

### 3.5 行数を選んで高さも埋める

行の割り方（＝行数 R と各行への配分）を変えると総高が変わる。**総高が rect.height を最も埋める割り方**を選ぶ：

- 実装候補（plan で確定）：**行数 R を探索**（`1..n`）。各 R でカードを document 順に**アスペクト和がなるべく均等**になるよう連続グループへ分割（＝行高が揃い、最後の行だけ極端に小さくならない）。各行に §3.3〜3.4 を適用し総高を算出。**総高が rect.height に収まる中で最大（最も埋まる）R** を採用。総高は R について概ね単調増加（R 大 → 行が高い → 総高増）なので二分探索可。
- 上限（§3.4）が全 R で効く少数カードのとき → どの R もカードは board 既定サイズ。最も board 的に見える割り方（幅を最大限使う小さめの R）を採り、余白は許容。

### 3.6 中央寄せ（残余の吸収）

- **水平**：幅を満たさない行（上限が効いた行・最後の部分行）は、その行だけ rect 内で水平中央寄せ。
- **垂直**：総高が rect.height に満たない残余は、ブロック全体を rect 内で垂直中央寄せ。
- 最終座標は前段どおり `rect.x / rect.y` を加えた**画面px絶対座標**（移動/リサイズ/回転はこの座標系のまま動く）。

## 4. 期待される結果

- **多数（100枚）**：full 行が幅を端まで満たし、行が縦に積まれて高さも埋まる → **充填率 ≈ 95%+**（現行 88%/77% から大幅改善）。右も下も端に届く。隙間はカードが小さい分だけ詰まる。
- **少数（数枚）**：上限（board 既定幅）で止まり、board-ish のサイズ・隙間で中央にまとまる（巨大化しない）。
- 残る余白は「最後の1行ぶんの小さな左右」＋「少数時の中央寄せ余白」だけ ＝ 現行のL字より遥かに小さい。

## 5. 契約（純関数 `fitSelectionToScreen`）

- **入力**：`cards: {id, width, height}[]`（縦横比のみ使用）、`rect: {x,y,width,height}`（安全領域）、任意 `opts?: { maxCardWidth?, gapRatio? }`（既定は §3.2 の BOARD_SLIDERS 由来値）。
  - 注：現行の第3引数 `gap`(px) は廃止（隙間は内部で比例算出）。呼び出し元 `handleEnterArrange` から `COLLAGE_GAP_PX` の受け渡しを外す。`COLLAGE_GAP_PX` は他に用途がなければ撤去。
- **出力**：`CollagePositions`（id → {x,y,w,h} 画面px絶対座標）。
- **不変条件（TDD で検証）**：
  1. 空 / 幅ゼロ / 高さゼロ → `{}`。
  2. 全カードに座標を返す（枚数一致）。
  3. 全カードが rect 内（`x ≥ rect.x`、`y ≥ rect.y`、右端・下端が rect を超えない・許容誤差内）。
  4. 縦横比保持：各カードの `w/h` が入力アスペクトに一致（許容誤差内）。
  5. 上限：どのカードの `w` も `maxCardWidth` を超えない（+ε）。
  6. 充填：均一な多数カードで**充填率が高い**（content が rect 面積の閾値以上を覆い、右端・下端に許容誤差内で到達）。現行 masonry より明確に高いこと。
  7. 隙間比：隣接カード間の隙間 ÷ カード高さ ≈ `gapRatio`（許容誤差内）。
  8. 少数カードで blow-up しない（max `w` ≤ maxCardWidth）かつ中央寄せ（左右に概ね等余白）。
  9. 回帰（N-40）：board 実既定（W=267.84 / G≈97.21 相当の比率）× 100枚 × 短く広い rect で**全カードが可視サイズ**（1px 崩壊なし）・rect 内。

## 6. 影響範囲（実装は plan で分解）

| ファイル | 変更 |
|---|---|
| `lib/share/collage-layout.ts` | `fitSelectionToScreen` を justified rows に**全面書き換え**（signature 変更：`gap` 廃止・任意 `opts`）。行の幾何・行数探索・上限・中央寄せの純ロジックを内包。`seedCollagePositions`/`moveElement`/`resizeElement`/`resizeElementFromCorner`/`bringToFront` は**不変**。 |
| `lib/share/collage-layout.test.ts` | `fitSelectionToScreen` の describe を**新契約に書き直し**（§5 の不変条件）。旧「倍率上限1＝自然サイズ維持」系の期待は破棄（新契約は縦横比のみ・board 既定を上限に充填）。他純関数のテストは温存。 |
| `components/board/BoardRoot.tsx` | `handleEnterArrange` の `fitSelectionToScreen(cards, rect, COLLAGE_GAP_PX)` → `fitSelectionToScreen(cards, rect)`。`cards` は従来どおり（内部で縦横比のみ使用するので変更不要）。 |
| `lib/board/constants.ts` | `COLLAGE_GAP_PX` を撤去 or 用途があれば残置（要確認）。`BOARD_SLIDERS` は不変。 |

**変えないもの（前段から流用）**：安全領域 `ARRANGE_SAFE_INSET`＋`.canvas`（盤面パネル内）／座標焼き込み（移動・リサイズ・回転は画面px座標）／`CollageCanvas` の描画・装飾追従（`--card-w`）・回転ノブ。

## 7. 検証

- **純ロジック**：TDD（§5 の不変条件を RED→GREEN）。行の幾何は閉形なので数値検証しやすい。
- **実機**：Playwright で content/rect 充填率が現行より大きく上がることを実測（1920×1080・1489×679・2560×1080 × 40/100枚）。到達手順は memory `reference_playwright_board_share_verify`（SELECT ALL→ARRANGE、IDB を preseed してオンボ/データ modal を回避）。ジェスチャ（移動/リサイズ/回転）は `setPointerCapture` で合成ポインタ不可 → ユーザー実機目視。

## 8. 非目標（YAGNI）

- 相対サイズの部分再現（大カードを背の高い行に置く等）は**やらない**（ユーザーが③を明確に放棄）。将来欲しくなれば別 spec。
- カードのクロップ／モザイク（treemap）は縦横比制約によりやらない。
- 自動配置後のジェスチャ仕様変更はしない（前段のまま）。
