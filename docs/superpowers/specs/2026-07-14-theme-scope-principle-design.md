# Design（方向性・親 spec）— テーマの適用範囲・原則の再確定（s196）

日付: 2026-07-14 / セッション 196
種別: **方向性（親）spec**。大物のため §7 で下記サブへ分解し、各サブが独自の詳細 spec→plan→実装を持つ（各 plan は安価モデルが写経できる粒度＝実 file:line・実コード・実コマンド埋め込み）。
関連: 本 spec は [2026-07-05-flat-theme-and-theme-boundary-design.md](2026-07-05-flat-theme-and-theme-boundary-design.md)（s162）の**決定#4「メニュー中立化」を意図的に更新**する（詳細 §6）。

---

## 1. 背景・動機

s162 で「テーマは盤面のみ・メニューは全テーマ中立」に決めた（サブ①＝メニュー中立化＋右ドロワー統一は s163 出荷済）。しかし s196 でユーザーが再検討を要望。核心は3つ:

1. **TUNE メニューの中身は今「オーディオミキサー」風**で、これは元々このアプリの既定＝**音（Sound Wave）テーマ**の顔として作り込んだもの。**捨てられないほど気に入っている**。だが紙・フラット・タワーに切り替えると世界観がズレる。
2. だから「TUNE にもテーマを乗せるべき」＝**chrome をテーマ化したい**。ただし TUNE だけ乗せて絞り込み/SETTINGS が中立だと"半分着替え"で最悪 → **chrome は全部乗せるかゼロか**が鉄則（s162 が中立に倒した理由でもある）。
3. TOWER（WebGL 看板テーマ）の実装を控え、「テーマが変える範囲」の**共通原則**を先に固めたい（ユーザー回答＝(c) 原則を決めたい）。

加えてユーザーは**スクロール方向（横・奥行き）がテーマで変わるのも面白い**と提案。奥行きの参考実装（無限ズーム・CSS transform ベース）を提供（[docs/private/2026-07-14-infinite-zoom-gallery-reference.md](../../private/2026-07-14-infinite-zoom-gallery-reference.md)）。

## 2. 確定した決定（s196 ユーザー合意）

### 2-1. テーマを「2層」で考える（本 spec の背骨）

- **【肌の層】= 全テーマが必ず設定する安い層**。色・font・明暗・背景/模様・スクロールメーター・カード登場/退場アニメ・**chrome（メニュー）の見た目**。トークン/設定で着替えるだけ＝**"抜け"（未対応で崩れる箇所）が構造的に起きない**。
- **【世界の層】= 凝ったテーマだけが opt-in する重い層**。スクロール方向（横/奥行き）・WebGL 背景・無限ズーム等、**盤面の挙動そのもの**を変えるもの。使わないテーマは触らない＝軽いテーマが重さに引きずられない。

### 2-2. chrome = (a)「1つの共通スキン系（トークン）＋任意アクセント」

- TUNE / 絞り込み / SETTINGS / 全パネルが**1つの共通スキン系（テーマトークン）で揃う**。**器は各自の最適形を維持**＝物理的に1箱には統合しない（**TUNE＝横型ホバー〔ユーザー選好・s172 で右ドロワーから差し戻した〕／SETTINGS・THEMES＝右ドロワー〔s163〕**）。色・font・質感・角丸・アニメ・hover 演出を**テーマのトークンから自動で着る**ので、器が違っても見た目が一貫する。
- ★調査で判明（s196）: 現状 chrome は**トークンを一切読まずダークガラス＋mono をハードコード**（s163 中立化の結果 `--chrome-btn-*` は死蔵）。オーディオミキサーは**全テーマ共通のハードコード既定**。→ 骨/皮化＝「各 chrome の直書きを `var(--chrome-*, 現行既定)` に置換し、テーマ側 `html[data-theme-id]` でトークン定義」。ミキサーは音テーマ（dotted-notebook）の皮へ格納し、未定義テーマは中立フォールバック。
- 各テーマは**任意で"署名的な飾り"を1枚**足せる（紙のロウ印のような）。土台は必ずトークンで着るので、**新テーマは色を渡すだけで chrome が完成＝抜けゼロ**。
- ＝作り込みは1回で全テーマに生き、テーマを増やしても崩れない。s162 の「中立固定」を、抜けを別の方法（骨/皮＋中立フォールバック）で防ぎつつ**表現を取り戻す**更新。

### 2-3. TUNE = 「骨」と「皮」に分ける／公開前に作り込む（甲）

- **骨（機能）= 共通**: TUNE が調整する項目（色・模様・格子の太さ・カード寸・角丸・タイトル font/サイズ 等）はどのテーマでも同じ＝**つまみが消える抜けが起きない**。
- **皮（見た目）= テーマごとに作り込む**: 音＝オーディオミキサー（**完成済・温存**）／紙＝紙・文房具の道具箱／タワー＝ビルの管理コンソール／フラット＝静かで清潔なパネル。
- **未作成テーマは"中立の皮"で自動フォールバック**（機能は動く・見た目だけ中立）。
- **公開方針＝甲（作り込んでから公開）**。最低ライン＝**音・Grid・紙・フラットの4テーマが世界観まで作り込まれている**こと（下 §5）。

### 2-4. 肌の層に含める項目（＝テーマで変わるもの・確定リスト）

既存の `ThemeMeta`（[lib/board/types.ts](../../../lib/board/types.ts) L53-96）が既にモデル化しているものが多い:

| 項目 | 既存の器 | 状態 |
|---|---|---|
| 背景・模様 | `backgroundClassName` + customization(pattern) | 既存 |
| 明暗（ライト/ダーク） | `colorScheme` | 既存 |
| カードの見た目（形・影・マット・破れ紙） | globals.css の `[data-theme-id]` + `decorations` | 既存 |
| カード登場アニメ | `motion.entry` | 既存 |
| カード退場アニメ | `motion.shutdown` | 既存 |
| カード文字の切替アニメ | `motion.text` | 既存 |
| スクロールメーター | `scrollMeterVariant`（waveform/ruler） | 既存 |
| 署名的な飾り | `decorations` | 既存 |
| 背景の巨大ワードマーク | themeId 連動（要整理） | 一部 |
| **chrome（メニュー）の font・文字色・アニメ・TUNE の出現と中身** | ❌ 今は中立固定 | **新規＝(a)で追加** |

### 2-5. 世界の層の候補（各々は専用 spec 送り）

- **TOWER**（超高層ビルの全窓＝カード・WebGL・レイアウト連動）＝既存 plan あり（[plans/2026-07-12-shader-theme-b-tower.md](../plans/2026-07-12-shader-theme-b-tower.md)）。共通 WebGL ランタイムをここで作る取り決め。
- **横スクロール本棚**（`ThemeMeta.direction='horizontal'` は既に型にあり、`InteractionLayer` に `isHorizontal` 分岐も存在＝操作の下地はある。レイアウト＝skyline は縦積みなので横用レイアウト変種が要る＝中規模）。
- **奥行き無限ズーム**（参考実装＝CSS transform ベース・framer-motion は `useRef` に置換で禁止回避・WebGL ではない）。**要設計判断**: 有限の実ブクマ（新しい順）を、有限ディープズームにするか無限散布にするか＝"これは私のブクマ"の所有感との両立。性能は 4K 実機で要実測（filter 不使用なので影より軽い見込み）。

## 3. surface 地図（テーマがどこまで乗るか・確定/推奨）

| 画面 | テーマの効き方 | 確度 |
|---|---|---|
| **盤面（本体）** | 肌の層フル＋（対応テーマは）世界の層 | 確定 |
| **chrome（TUNE/絞り込み/SETTINGS）** | (a) で乗せる＝テーマの皮 | 確定 |
| **共有の受け取り画面（/s/）** | **盤面と同じ＝肌の層＋世界の層まで乗せる**（TOWER のビルも相手の画面に立つ）。＝作った人の看板が見た人の画面でも立つ | **推奨（レビューで最終確認）** |
| **ライトボックス（カード拡大）** | テーマ非依存（暗いバックドロップで固定） | 確認（現状維持想定） |
| **共有画像（書き出し1枚）** | 盤面の見た目ごと写る（TOWER 等はスナップショット） | 確定（既存） |
| **トップページ（LP）** | 盤面テーマ非依存（白エディトリアルで固定） | 確定（既存） |
| **拡張機能の設定画面（options）** | **テーマ非適用＝フラットで分かりやすい中立メニュー**（現状はサウンド波形＝SIGNAL/オシロスコープの世界観が乗っている → 剥がす）＋**多言語化**（現状ハードコード英語 → 読む人の言語で） | **確定（ユーザー要望 s196）** |

**受け取り画面（推奨の理由）**: 本アプリのミッション＝「見た人に広めてもらう看板」。TOWER を共有→相手が暗い普通の盤面を見るだけでは看板が消える。受け取りも"盤面"だから同じテーマを立てる。**正直なコスト**: 相手の端末が不明なので世界の層（WebGL 等）は**フォールバック必須**（WebGL 不可→暗い盤面／低性能端末→静的背景）。実装は TOWER 側と受け取り側の両方に少し広がる。

## 4. 拡張設定（options）の扱い（ユーザー要望 s196・確定）

現状（実コード確認済）: [extension/options.html](../../../extension/options.html) はサイドバー＋波形オシロ（`SIGNAL/LIVE`・`SYNC`・`MOTION`）＝**音テーマの世界観**。テキストは全て**ハードコード英語**（i18n 無し・`<html lang="en">`）。`extension/_locales/` は **en のみ**。

- **(i) 脱テーマ・フラット化**: SIGNAL/オシロスコープ等のサウンド演出を外し、**中立でわかりやすいフラットな設定メニュー**へ。盤面のテーマとは連動させない（拡張は"道具"の面＝§3 の原則どおり中立）。
- **(ii) 多言語化**: Chrome 拡張の標準 `_locales/<lang>/messages.json` 方式で15言語。HTML/JS の文言を `__MSG_key__` / `chrome.i18n.getMessage` に置換。表示言語はブラウザ UI 言語に追従（`chrome.i18n` の既定）。
- **不変則**: 機能名（AllMarks/SETTINGS/SHORTCUTS 等）は訳さない（globally-clear English 方針）／説明文は母国語（i18n）。盤面 i18n（messages/*.json）とは**別系統**（拡張は Chrome i18n）＝混ぜない。

## 6. s162 spec との関係（何を更新し何を残すか）

- **更新**: s162 決定#4「メニュー類＝全テーマ共通の中立 chrome（テーマは一切乗らない・例外なし）」→ **本 spec で "chrome はテーマの皮を着る（骨/皮＋中立フォールバック）" に置き換え**。s163 サブ①で作った「右ドロワー統一・中立化」の**構造（右ドロワー・機能骨）は土台として活かし**、その上に per-theme の皮を載せる（作り直しではなく皮の追加）。
- **残す**: s162 決定#2「音（Sound Wave）テーマの盤面アイデンティティは温存」／決定#3 の肌5項目（本 spec の肌の層に吸収・拡張）／フラット default テーマ（s162 サブ②が本 spec §5 のフラットに対応）。
- **超える**: s162 は「盤面のみ・世界の層という概念なし」。本 spec は世界の層（方向/WebGL/奥行き）と受け取り画面への拡張を足す。

## 7. 分解と順序（各サブが独自 spec→plan→実装・plan は安価モデル写経粒度）

> 原則: 肌の層＝安全なので先に基盤化。世界の層＝TOWER を先頭に共通ランタイムを作る（既存 plan）。公開ラインナップ（§5）を最短で満たす順に。

- **サブ1【基盤・最優先】chrome の骨/皮化＋中立フォールバック**: (a) を実現する土台。①chrome を駆動するテーマトークン（色/font/角丸/アニメ）の定義②TUNE/絞り込み/SETTINGS を"骨（機能）＋皮（テーマ別スキン）"構造に③皮未定義テーマは中立スキンで動く④音テーマの皮＝現ミキサーをこの構造に移植（見た目バイト維持）。**"抜けゼロ"の検査（全テーマ×全パネルが必ず描画される）をテストで固定**。
- **サブ2 フラットパターンテーマ＋その TUNE 皮**: s162 サブ②の白フラット default を本 spec の枠で作る（`kind:'pattern'`・静かなメーター/モーション・清潔な TUNE 皮）。一般ユーザーの標準。
- **サブ3 Grid・紙の TUNE 皮を作り込む**: サブ1 の骨/皮に、Grid＝方眼世界／紙＝道具箱の皮を彫る。紙は `work`（customization 無し）だが chrome の皮は着る。
- **サブ4【世界の層・先頭】TOWER**: 既存 plan（shader-theme-b-tower）。共通 WebGL ランタイム＋レイアウト連動をここで作る。**受け取り画面のフォールバック方針（§3）をこの plan に取り込む**。
- **サブ5 受け取り画面をテーマ完全化**: 世界の層（TOWER 等）を /s/ にも立てる＋端末別フォールバック。§3 推奨をレビュー確定後に。
- **サブ6【独立】拡張設定の脱テーマ・フラット化＋15言語 i18n**: §4。拡張コードベース（`extension/`）内で完結・盤面と別系統。TOWER 等と無関係なので**いつでも並行可**。
- **公開後（世界の層の追加）**: 横スクロール本棚 / 奥行き無限ズーム。各々専用 spec（奥行きは §2-5 の設計判断＝有限 vs 無限を先に詰める）。

## 8. 不変条件・リスク（全サブ共通で死守）

- **抜けゼロの保証**: 肌の層は全テーマがトークンで着る＝どのテーマでも chrome/メーター等が欠けない。テストで「全テーマ×全 chrome パネルが描画」を固定。
- **性能（4K/258% 実機）**: s196 で紙テーマの三重 drop-shadow が 4K でフレーム落ちした前科（N-61）。世界の層（WebGL/無限ズーム）は DPR 上限・パス数・unmount 規律を守る。肌の層のアニメも 4K で測る。
- **Framer Motion 禁止**（CLAUDE.md）: 奥行き参考の framer-motion は `useRef` へ置換。GSAP+CSS keyframes 方針を維持。
- **デスクトップ既定盤面のバイト同一は"意図的に更新"される**（s162 と同じ扱い＝chrome がテーマ化するので中立時のバイト同一は保証対象外。ただし各サブは「音テーマ選択時の盤面アイデンティティ」は温存）。
- **保存値に書かない**: 世界の層のレイアウト連動（TOWER の窓格子・横/奥行き）は表示時 override のみ（MOBILE_LAYOUT の掟）。テーマを戻せば元通り。
- **i18n 不変則**: 盤面（messages/*.json）と拡張（_locales）は別系統。placeholder 保持・機能名は訳さない（s196 C0 の placeholder-parity テストが盤面側を守る）。

## 9. 各サブで詰める未確定（本 spec は方向のみ）

- サブ1: chrome トークンの具体セット（どの CSS 変数を"皮"が読むか）・中立スキンの視覚言語・音ミキサー皮の移植で現行バイトをどこまで維持するか。
- サブ2: フラットの具体パレット（LP トークンとの対応）・静かなメーター/モーションの定義。
- サブ4/5: 受け取りの世界の層フォールバック段階（WebGL 不可→静的背景の絵）。
- 奥行き: 有限ディープズーム vs 無限散布（所有感との両立）・撮影（SHARE）での切り出し。

## 10. 非対象（本 spec に含めない）

- **収益・支援・価格・ローンチ戦略**＝機微につき `docs/private/` のみ（本 tracked spec には一切書かない）。支援ページの早期化（公開直後から支援を受けられる状態）は非公開のローンチ計画（release-runway 束D2 ほか）で扱う。
- 各テーマの**具体的なビジュアルデザイン**（モック）＝各サブ spec でユーザー承認を取ってから。
