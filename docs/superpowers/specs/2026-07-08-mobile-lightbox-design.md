# スマホ専用ライトボックス 設計 (2026-07-08 / セッション180)

## 背景と現状

- スマホ盤面は s178–s180 で完成（全画面3列グリッド・ボトムナビ・ネイティブスクロール）。カードタップで `Lightbox` が開くが、**`Lightbox.tsx`（2263行）に `isMobile` 分岐が皆無** → デスクトップ用の2カラム（左=メディア／右=テキスト、`min(94vw,1240px)` 枠）がそのままスマホに出ており、小画面で窮屈。
- 本 spec はこのライトボックスを**スマホ専用の没入型UI**に作り替える。memory `project_mobile_board_direction` の "mobile-specific Lightbox pending" に対応。

## ゴール / 非ゴール

- **ゴール**: スマホ（`isMobile` = `max-width:640px`）で、タップしたカードの主役（画像/動画/ツイート/文字）を**画面中央にできる限り大きく**表示し、4方向ジェスチャ（左右=送り／上=情報／下=閉じる）で操作できる没入型ライトボックス。開閉はPCと同じモーフ（タップ位置から拡大→中央大、閉じは元スロットへ縮小着地）。
- **非ゴール(A)**: デスクトップのライトボックスの挙動変更。**PC経路は1pxも変えない**（全て `isMobile` ゲート）。
- **非ゴール(B)**: 盤面側の変更（s180で完了済み）。
- **非ゴール(C)**: 640px超（タブレット）の専用対応。今回はスマホのみ。

## 確定した操作モデル（ユーザー承認済み）

タップで主役を中央に大きく表示。以降、指の4方向で操作（**軸ロック**＝指を置いて最初の有意な移動で縦/横どちらの操作かを確定し、離すまで固定）:

| ジェスチャ | 動作 |
|---|---|
| **タップ**（盤面カード） | ライトボックスを開く（モーフで拡大） |
| **左右スワイプ** | 前後のカードへ送り（横スライド遷移。`nav.onNav(±1)` に接続。端は軽い抵抗で止まる） |
| **下スワイプ** | 閉じる（指追従：主役が下がり少し縮む→閾値超えで離すと閉じ＝モーフで元スロットへ／戻すと復帰） |
| **上スワイプ** | 情報シートをせり上げ（タイトル・説明・出典・メタ・翻訳等） |
| **主役以外/シート外のタップ** | 閉じる |
| **複数画像の切替** | 画像ドットの**タップ**（左右スワイプはカード送りに使うため競合回避。既存ドット流用） |

- PC用の前後チェブロン矢印（`LightboxNavChevron`）は**モバイル非表示**（タッチはスワイプで送る）。

## レイアウト

- **暗幕**: 既存 `backdrop`（z100、dim + outside-tap-to-close）を流用。
- **主役（中央大）**: 画面中央にメディアを `contain` で最大化。chrome・余白は最小。カード種別で中身が変わる（下記）。この主役要素に**既存の `mediaRef` を付ける**（モーフの着地先になるため。後述）。
- **情報シート**: 画面下端に隠れており、**上スワイプ**または下端の**薄いグラブハンドル（つまみ）タップ**でせり上がる。中身はタイトル・説明・出典リンク・メタ（host等）・ツイートは翻訳トグル。**発見性のためグラブハンドルは常時薄く表示**。下スワイプ/ハンドル再タップで戻す。シートが出ている間、主役は少し縮小＋暗転して背後に控える。
- **閉じるボタン（✕）**: スマホでも右上に小さく置く（下スワイプを知らないユーザーの保険）。

## カード種別ごとの主役と縦スクロールの扱い

縦ジェスチャ（上=シート／下=閉じる）と、主役内部の縦スクロールが競合し得る。**ルール: 縦ドラッグはまず主役内部がその方向にスクロール可能なら内部スクロールに使い、スクロール端に達して初めて 閉じ(下)/シート(上) が発動**（iOSのボトムシート標準・既存の `[data-card-scroll]` 端判定と同じ発想）。

| 種別 | 主役（中央大） | 情報シート | 内部スクロール |
|---|---|---|---|
| 画像（単/複数） | 画像を大きく中央（複数はドットで切替） | タイトル・説明・出典・メタ | なし（画像は縦ドラッグ即 閉じ/シート） |
| 動画/YouTube/TikTok | プレイヤーを大きく | 同上 | なし |
| ツイート | 本文＋メディアを縦に（`TweetColumns` の中身を再利用） | 出典・メタ・**翻訳トグル** | 本文が長ければ縦スクロール。端で 閉じ/シート |
| 文字カード | 本文を大きく読みやすく中央 | 出典・メタ（最小） | 本文が長ければ縦スクロール。端で 閉じ/シート |

## 開閉アニメ（PCと同じモーフを流用）

- PCの実体（[Lightbox.tsx:993-1058](../../../components/board/Lightbox.tsx#L993-L1058)）は**クローン方式**（R3Fシーンは `SCENE_ENABLED=false` で無効）: タップしたカードの `cloneNode` を body 直下に持ち上げ、**`width/height/top/left` を「カードの実測rect（`sourceRect`）」→「`.media`（`mediaRef`）の実測rect」へ補間**。クローン飛行中は `.media` を `opacity:0` で待たせ、着地フレームで入れ替え（ハンドオフ）。`transform:scale` を使わないので角丸がGPU再サンプルされない。
- **モバイル流用の要点**: 上記コードは**着地先を `mediaRef.current.getBoundingClientRect()` で実測**する。よってモバイルの主役（中央大表示）に `mediaRef` を付ければ、**同じコードがそのまま中央大へ向かって拡大**する。閉じは逆再生で**元スロットへ縮小着地**。→ 「PCと同じアニメ」がコード流用で実現。
- **文字カードのフォールバック**: モーフはクローン元にサムネ相当が要る（画像/動画/ツイートは可）。文字カードはサムネが無いので**シンプルな中央拡大フェード**にフォールバック（PCと同挙動。memory `project_lightbox_flip_content_equivalence` の文字カードFLIP非等価の既知事項）。
- **カード送り（左右スワイプ）はモーフではなく横スライド**: `view` を隣カードに差し替え、主役を translateX で送り出し/送り込む別トランジション。モーフは「開く/閉じる」専用、スライドは「送り」専用と役割分離。

## アーキテクチャ（分離と再利用）

`Lightbox.tsx` は既に 2263 行で肥大。モバイル分岐を同ファイルに詰めず、**独立コンポーネントに切り出す**（brainstorming「1ユニット1責務」）:

- **`MobileLightbox.tsx`（新規）**: モバイルの見た目（主役中央大＋情報シート＋✕）。`Lightbox` は `isMobile` の時これを描画し、PC時は現行の `.frame` 2カラムを描画。主役の中身は**既存の `LightboxMedia` / `DefaultText` / `TweetColumns` の内部を再利用**（描画の枠だけモバイル用に置換）。`mediaRef`・`textRef`・`nav`・`requestClose`・`view`・`originRect`/`sourceCardId` は現行の Lightbox から props で受け渡し、**モーフ機構（Lightbox側の effect）は無改造**で機能させる（`mediaRef` が主役に付く前提）。
- **`use-lightbox-swipe.ts`（新規フック）**: 4方向パンの軸ロック・閾値・指追従 transform・完了/復帰判定を担う。**閾値ロジック（純関数部）は vitest でユニットテスト可**、実タッチの感触は実機のみ。
- **`LightboxInfoSheet.tsx`（新規）**: 下端の情報シート（上スワイプ/ハンドルで開閉）。中身は再利用したテキスト/メタ/翻訳トグル/画像ドット。
- 既存の `LightboxNavChevron`・PC用 `.frame`/`.media`/`.text` レイアウトCSSは**モバイルで不使用**（描画しない）。

### ジェスチャ閾値（初期値・実機で調整）

- 軸ロック: 最初の移動が |Δx|>|Δy| なら横（送り）、縦なら上/下。しきい 8–10px で確定。
- 閉じ（下）: 変位 > 画面高の25% **または** 下向きフリック速度超過で確定。未満は spring 復帰。
- 送り（左右）: 変位 > 画面幅の35% または横フリックで確定。端カードはラバーバンド抵抗で止める。
- シート（上）: 変位 > 一定 or 上フリックで全開。半端は開/閉どちらか近い方へ。

## 検証戦略（s180の教訓を厳守）

- **ジェスチャ・スクロール・モーフの体感は実機のみで検証**（Playwrightの合成タッチ/JSスクロールは `touch-action` と実タッチ挙動をすり抜ける＝memory `reference_native_scroll_touch_action_playwright`、`project_mobile_board_direction` のCDP合成タッチ1回問題）。
- **Playwright + tsc + vitest** は「構造（DOM・isMobile分岐でモバイル要素が出る/PC要素が出ない）」「スワイプ閾値の純関数ロジック」「デスクトップ回帰ゼロ（1489で従来通り2カラムが出る）」を担保。
- 各段階で本番 `allmarks.app` にデプロイ→ユーザー実機確認をゲートに置く（fast per-feature の実機確認）。

## リスク / 未決（実装時に潰す）

1. **モーフの mobile 実挙動未検証**（memory `reference_lightbox_morph_gotchas` / `reference_lightbox_hires_flip_caching` の既知の罠：`:root` ミラー・cloneNode が video の muted を落とす・hi-res が FLIP 時に極小描画）。まず画像カードで疎通確認してから展開。
2. **主役内スクロール端 ↔ 縦ジェスチャの受け渡し**（ツイート/文字カードのみ）。端判定は既存 `[data-card-scroll]` の overflow/atBottom を流用。
3. **情報シートの発見性**（グラブハンドルの見せ方）。薄い常時表示＋初回のみ微アニメで誘導を検討。
4. **動画/ツイートの再生**（Tier系の inline 再生はライトボックスでは別扱い。現行PCの `LightboxMedia` 再生を踏襲）。
