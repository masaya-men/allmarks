# SHARE アレンジ「1画面に最大サイズで自動配置」設計書（N-40 ＋ N-41 回転ノブ刷新）

- 日付: 2026-07-06（セッション167）
- 親 spec: [2026-07-06-share-collage-screenshot-rebuild-design.md](2026-07-06-share-collage-screenshot-rebuild-design.md)（SHARE 作り直し全体）
- 対象バグ: **N-40**（アレンジで多数カードが出ない）／同時に **N-41**（回転ノブの見た目を業界水準に）

---

## 1. 背景・問題（root cause 確定）

SHARE の第2段「並べる（アレンジ）」は、選んだカードを**自由配置キャンバス `CollageCanvas`** に置き、ユーザーが自分の OS のスクショ（Win+Shift+S / ⌘+Shift+4）で範囲選択して共有する仕組み。**アプリ内の画像化ボタンは無い**（他サイトのサムネ＝クロスオリジン画像が混ざると画像化が黒く失敗する既知の技術壁のため、手動スクショ方式）。→ 出力は常に**「今見えている1画面」**。

s166 で「盤面そっくり（WYSIWYG）」にするため、`handleEnterArrange`（[BoardRoot.tsx:2016](../../../components/board/BoardRoot.tsx#L2016)）が**各カードを盤面の実座標**に置くようにした。ところが `CollageCanvas .root` は `position:absolute; inset:0`（[CollageCanvas.module.css:5](../../../components/board/CollageCanvas.module.css#L5)）＝**1画面固定でスクロールしない**。結果、たくさん選ぶと縦長の盤面コピーになり、**画面下にはみ出したカードは「見えない・触れない・スクショにも写らない」**。これが「多数のカードが出ない」の正体。

## 2. 確定した方針（ユーザー承認済み）

> **選んだカードを、何枚でも・大きさがバラバラでも、1画面に「できるだけ大きく」自動で詰めて全部収める。その後ユーザーがつまんで移動・拡大・回転して仕上げてスクショ。**

- **スクロールさせない**。画面外に置かない（N-40 を原理的に解消）。
- **隙間を消して最大サイズ**で詰める（盤面の“元の間隔”は再現しない＝コンパクトに寄せる）。
- キャンバスに**ズーム/パン機能は足さない**（シンプル最優先）。「1画面に最大で収める＋つまんで微調整」だけ。
- 既存の自由配置・隅リサイズ・自由回転・見出し（タイトル）・紙テーマ装飾は**そのまま維持**。
- **N-41 回転ノブ刷新を同じアレンジ画面内で同時に**片付ける（見た目のみ）。

### 非対象（out of scope）
- ズーム/パンできるキャンバス（将来案）。
- アプリ内の画像書き出し（`/s` の COPY LINK＝phase3 の別作業）。
- 受け取り側 `/s` の変更。盤面グリッド本体の変更。

## 3. 挙動仕様

### 3.1 入力
- 選択カード集合（`selectedIds`）を**盤面順**（`lightboxNavItems` の順＝新しい順）で並べたもの。
- 各カードの**自然サイズ**: 幅 = `customWidths[id] ?? cardWidthPx`、高さ = `itemSkylineHeight(item, w)`（盤面 skyline と同じ算出）。

### 3.2 安全領域（safe rect）
アレンジで使える矩形を「画面全体」ではなく**上下左右の chrome を避けた安全領域**にする。少なくとも：
- 上: 上部クロム（ヘッダー等）に潜らない余白（既存 `BOARD_TOP_PAD_PX` 相当を土台に、アレンジ用インセット定数を新設）。
- 下: `ShareToast`（下部バー・z:116）に隠れない余白。
- 左右: 端に貼り付かない余白。

この safe rect の内側に**全カードが収まる**ことを保証する（s165 で cosmetic 既知だった「初期カードが上部クロム裏に少し潜る」も同時に解消）。

### 3.3 アルゴリズム（1画面に最大サイズで詰める）
純ロジックとして切り出す（TDD）。概略：

1. `containerWidth = safeRect.width` で、カードを**自然サイズのまま** `computeSkylineLayout` で詰め、`totalHeight` を得る。
2. これが `safeRect.height` に収まれば**倍率 s = 1**（縮小なし＝盤面と同じ大きさ）。収まらなければ、**倍率 s（0 < s ≤ 1）を二分探索**して「詰めた高さが safeRect.height に収まる**最大の s**」を求める。
   - s を大きくするほどカードが大きく → 1行に入る枚数が減り → 縦が伸びる（≒単調）。よって二分探索で「収まる中で最大」が取れる。
   - **横幅は常に safe rect いっぱい**を使う（＝ムダな左右余白でカードが小さくなるのを防ぐ）。
3. 求めた s を各カードの `x/y/w/h` に**焼き込む**（＝画面ピクセルの実サイズにする）。これで既存の移動/リサイズ/回転（全て画面px基準）が**一切の変換なしでそのまま動く**。
4. 詰めたブロックを safe rect 内に配置（左上基準、縦に余りがあれば中央寄せは任意）。
5. **自動配置は 80px 下限（`COLLAGE_MIN_WIDTH_PX`）を無視してよい**（80px 下限はユーザーが手で縮める時のガードであって、自動配置には適用しない）。100枚などで 80px を割っても全部収めることを優先。

### 3.4 枚数による見え方
- **数枚**（自然サイズで1画面に入る）→ s=1 ＝**盤面と同じ大きさ**（s166 の忠実さを維持）。余白は空くが問題なし。
- **たくさん（〜100枚）**→ s<1 で**忠実に縮小**して全部収める。100枚なら1枚≈100px前後の小さめサムネ。全部見える・全部スクショに写る（1画面を100分割する物理的帰結。選択上限は 100 枚）。

### 3.5 自動配置後の操作（不変）
- つまんで移動 / 隅リサイズ（対角投影）/ 自由回転（15°磁気スナップ）/ タイトル編集・移動・拡縮 / 掴んで最前面 —— **すべて現行のまま**。
- RESELECT→ARRANGE の再入場は**まっさらな配置で再シード**（回転も {} にリセット。現行踏襲）。

### 3.6 数枚時の上限（確定：キャップ1）
- **s の上限は 1（自然サイズ）にキャップ**＝数枚しか選ばなくても盤面と同じ大きさで止め、画面いっぱいに膨らませない。大きくしたければユーザーがドラッグ。s166 の“盤面そっくり”を尊重。**ユーザー承認済み（s167）**。

## 4. N-41 回転ノブ刷新（見た目のみ）

- 対象: `CollageCanvas` の回転ハンドル DOM（`.rotateHandle` 内の `.rotateStem`＋`.rotateKnob`）と CSS。
- 現状: 上中央に浮く「細い線＋白丸12px」＝業界水準でない（ユーザー指摘）。
- 変更: **Canva/Figma 風の回転アイコン**（弧＋矢印の“くるっと回る”グリフ、SVG）に差し替え。カード上でも視認できるよう小さな円形の背景（半透明ダーク等）に載せる。**hover で出現**（静止＝スクショ時は非表示）は維持。
- **不変**: 角度ロジック `lib/share/collage-rotate.ts`（`pointerAngleDeg`/`rotateFromPointer`）、配線 `handleRotatePointerDown`、ハンドルの当たり判定位置（上中央）、hover 表示挙動。**見た目（マークアップ＋CSS）だけ**を変える。

## 5. データ・永続

- アレンジの配置・回転・タイトルは**一時 React state のみ**、IDB に**永続しない**（現行契約を維持）。DONE/CANCEL/Esc/RESELECT で破棄・再シード。盤面本体（グリッド・並び順・サイズ）は**一切変更しない**。

## 7. テスト観点

**純関数（vitest・TDD）**
- 全カードが safe rect 内に収まる（右端・下端がはみ出さない）。
- 収まる最大サイズ：s を上げると収まらなくなる境界＝返り値が「収まる中で最大」。
- サイズ混在（大小バラバラ）でも重ならず収まる。
- 1枚 / 0枚 / 100枚 の端。100枚でも全部 rect 内（80px 下限を割ってよい）。
- 数枚で自然サイズに収まる → s=1（座標が自然パックに一致）。

**Playwright（out/ ローカル＝デプロイ成果物、preseed IDB でオンボ回避）**
- SELECT ALL → ARRANGE で `collage-el-*` の数 == 選択数（**取りこぼしゼロ**）。
- すべての `collage-el-*` の bounding rect が**ビューポート内**（画面外に無い＝N-40 解消の実証）。
- 多数選択（例: 30〜100枚を preseed）でも上記が成立。
- 参考 memory: `reference_playwright_board_share_verify`（SELECT ALL+ARRANGE 到達手順、IDB 事前投入）。

**手動目視（ジェスチャは `setPointerCapture` で合成ポインタ不可）**
- 移動/リサイズ/回転/タイトル編集が自動配置後も自然に効く。
- 回転ノブの新デザインの見た目・視認性・hover 出現。
- 本番 `allmarks.app` ハードリロードで実機確認。

## 8. リスク・留意

- **100枚 = カードが小さい**のは物理的帰結（1画面÷100）。仕様として受容（上限100枚）。忠実な大判1枚が欲しい欲求は phase3 COPY LINK（`/s`）の役割と役割分担。
- **多数の実カード面を静的描画**（video/tweet 含む）：盤面は既に100+枚描画実績、アレンジは選択分のみ＋静的（autoCycle/ambientOn off）なので性能懸念は低いが、100枚時のスクショ前レイアウト確定を実機で確認。
- **s166 の WYSIWYG“完全一致”は数枚時のみ**（多数は忠実縮小）。ユーザー合意済み（「収まる中で最大」を優先）。
- 回転ノブは**見た目のみ**変更＝角度挙動のリグレッション無しを既存テストで担保。
