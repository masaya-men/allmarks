# LP 作り直し — 設計書 (2026-06-17)

> フェーズ1 = LP のデザインと中身 (英語 + 日本語) を作り直す。
> フェーズ2 (= 多言語 URL / hreflang / 言語別 sitemap = i18n 層②) は **本設計の対象外**。LP の構造が固まってから別 spec で被せる。

---

## 0. ゴールと非ゴール

**ゴール**
- 既存 LP (平凡と判断) を、白基調・シンプル・洗練・編集的タイポ・スクロール演出の LP に作り直す。
- ミッション「整理ツールではなく**表現ツール**」を伝え、ボード(黒)へ地続きで誘導する。
- 英語(既定)+ 日本語の 2 言語で原稿を用意する。

**非ゴール (今回やらない)**
- 多言語 URL ルーティング (`/ja` `/zh` …)、hreflang、言語別 sitemap = **層②は次回**。
- 残り 13 言語の翻訳 = 次回。今回は en/ja のみ。
- マーケ下層ページ (features/guide/faq 等) の中身刷新 = 今回はリンクの導線維持のみ、刷新は別タスク。

## 1. ビジュアルシステム (LP 全体の見た目の文法)

| 要素 | 決定 |
|------|------|
| 下地 | 純白〜ごく淡いオフホワイト (`#FAFAF8` 系)。インク = 黒に近いチャコール |
| アクセント | **緑チェック `#28F100` の 1 色のみ**。リンク・小さな印・ホバー・スクロール進捗など「点」で効かせる。多色はコラージュ素材側で見せ、地の配色は禁欲的に |
| 大見出し | 品のある欧文セリフ (雑誌表紙の風格)。日本語見出しは品のある明朝(性能と相談、§3b) |
| 本文・UI | Geist (既存同梱) |
| 文字組 | 字間・ウェイト・余白で勝負。派手な装飾に頼らない |
| 奥行きの背骨 | 巨大なセリフのキーワードが**背景層**としてゆっくりパララックス、**手前のコラージュカード層**が速く動いて文字を一部隠す → 奥行き。ボードのワードマークそのままの転用ではなく、`frontend-design` スキルのルールに沿って LP 用に洗練させた多層パララックスとして再構成 |
| 終端の暗転 | 最終 CTA 直前で白 → 黒へスッと遷移。「ここから先が作品づくりの場」。アプリ(黒)へ地続き |
| モーション原則 | 全要素がスクロールで現れる (フェード + わずかな上昇 + パララックス)。動きは静か・上品。`prefers-reduced-motion` で全部おとなしく (`gsap.matchMedia`) |

**重要な制約**: 音波モチーフは**デフォルトテーマに過ぎず、今後テーマを増やす**。よって LP の見た目を音波に依存させない。LP のビジュアルはテーマより上の「ブランドの層」(白基調・編集的タイポ・A ロゴ + 緑チェック)で作る。音波は終端付近の演出に薄く香らせる程度に留める(必須ではない)。

## 2. スクロールの物語 (セクション構成)

**user 採用コピー方針 (2026-06-17 改訂)**: 余計なポエム・装飾表現を排し、ミニマルかつ機能美が伝わる文体。9 セクション案を user 提案で **6 ブロック**に整理。FEATURES は番号付き (01–05) だが、**よくある SaaS の箱グリッドにはせず**、白い余白 + 奥行きタイポ + スクロールで 1 つずつ立ち上がる編集的な見せ方にする (テンプレ感回避、`.claude/rules/ui-design.md`)。FEATURES の英語ラベル (CAPTURE/LAYOUT/LIVE GRID/ORGANIZE/PRIVACY) と番号は全言語で英語固定 (UI 英語語彙、翻訳対象は title/body のみ)。

| # | ブロック | 役割 / 伝える情報 | 演出 |
|---|-----------|------------------|------|
| 1 | **HERO (白)** | ラベル `Visual Bookmark Manager` / 主見出し「リンクを、ビジュアルボードに。」/ 説明 (ワンクリック保存・自由コラージュ・登録不要/完全無料) / ボタン「ボードを開く」「使い方を見る」 | 大きなセリフ見出し、背後に奥行きタイポ、手前に厳選コラージュカードが数枚静かに浮かぶ |
| 2 | **PROBLEM** | 「溜めるだけで見返さない青いリンクの羅列 / スクロールして終わるタイムライン」を機能的に一刺し | 余白で"間"を作る。少ない要素 |
| 3 | **FEATURES** (5 機能を 1 ブロックに集約、編集的シーケンス) | **01 CAPTURE** どこからでもワンクリック保存 / **02 LAYOUT** 自由なコラージュ配置 / **03 LIVE GRID** 複数動画を同時再生 (実際に動く) / **04 ORGANIZE** タグで整理・テーマで着替え (テーマは今後増える) / **05 PRIVACY** 登録不要・ローカル完結 | 番号付きだが箱グリッドにしない。01–05 が白い余白の中でスクロールで順に立ち上がる。03 LIVE GRID で実際に動画再生 |
| 4 | **SHARE** | 「1 枚のボードとして共有」。画像エクスポート + リンク共有 (配置のまま相手に届く) | — |
| 5 | **FINAL CTA (暗転)** | 白 → 黒へ。「さあ、ボードを作ろう。」/ ボタン「今すぐ始める（登録不要）」。アプリ (黒) へ地続き | 背景色遷移 (ScrollTrigger) |
| + | **フッター** | 機能/ガイド/FAQ/About/プライバシー/規約/問い合わせへのリンク。言語切替の置き場は次回 (層②) | — |

**FEATURES 03 LIVE GRID** = §3a の動画方針 (ロイヤリティフリーのループ動画を複数同時 + 本物 YouTube を `YouTubeEmbed` のミュートループモードで) をここで実装。**PRIVACY が FEATURES の締め**として安心を置き、SHARE → CTA へ。

## 3. 素材・技術・原稿

### 3a. デモ素材

- 厳選した **CC0 画像** を `public/marketing/` に最適化 WebP で同梱 (約 12〜20 枚、多色・多様式、ダサいものは入れない)。
  - パブリックドメインの名画: メトロポリタン美術館 Open Access / シカゴ美術館 / アムステルダム国立美術館 (すべて CC0)
  - CC0 写真: Unsplash / Pexels (商用利用自由・帰属表示不要)
- 出どころと各画像のライセンスを `docs/` にメモして証跡を残す。
- 製品の実画面 (セクション 4・5) は**本物のボード部品を流用**し、中身だけこの安全素材に差し替え。個人情報ゼロで「本物の製品」に見せる。
- **ユーザーの実ボードのスクショは使わない** (公開ページに個人ブクマが出るため。プライバシー原則)。
- **動画素材 (セクション 5、必要ならセクション 4 にも 1 枚)**: ロイヤリティフリー / CC0 の短いループ動画 (Pexels Videos / Coverr / Mixkit、商用利用自由) を厳選。**実際にミュート自動再生でループさせ、動いている様子を見せる**。短く (≤10s)・小さく最適化 (≤720p, 低ビットレート, mp4 + 可能なら webm)・ポスター画像付き・遅延読み込み・画面内に入った時だけ再生。同時再生は数本に抑える。`prefers-reduced-motion` ではポスター静止のみ。
- **本物の YouTube (ハイブリッド、user 承認)**: 権利クリーンなチャンネル (NASA = パブリックドメイン等、公式機関のみ。特定の個人クリエイターは推薦誤解を避け不使用) の実 YouTube を、**製品の実コンポーネント `YouTubeEmbed` のミュート・ループ・プレーヤーUI非表示モード (`controls=0`+`modestbranding`)** で自動再生。「本物の動画も実際に再生される」誠実さを担保しつつ白い洗練トーンに溶かす。「保存」「並べる」セクションには本物の YouTube サムネカードも 1〜2 枚。性能のため iframe は最小限・画面内のみ・restricted 時はポスターにフォールバック。

### 3b. 技術と性能 (プロジェクト規約準拠)

- **既存スクロール基盤 (Lenis = `useSmoothScroll` + GSAP ScrollTrigger = `useScrollTrigger`) はそのまま流用**。`LandingPage.tsx` を司令塔に残し、中の 6 セクション部品 (Hero/Save/Collage/StyleSwitch/Share/Cta) を新しい 9 セクション構成に作り直す。
- **Vanilla CSS Modules** (Tailwind 不使用) + CSS 変数でデザイントークン管理。**Framer Motion 不使用、GSAP のみ**。
- パララックス・暗転は **transform と不透明度だけ**で動かす (レイアウト揺れを避け 60fps 維持、`gsap-performance`)。`prefers-reduced-motion` は `gsap.matchMedia` で全部静かに。
- **フォント**: 欧文セリフ見出しは Geist と同様に**自己ホスト (next/font) で 1 書体**追加。日本語見出しの明朝は重くなりやすいので、**見出し限定で字数を絞って使う** or 上品なフォールバックに留める(性能優先、実装時に実測判断)。具体書体は実装時に `frontend-design` スキルの指針で選定 (汎用フォント禁止)。
- 性能目標は既存どおり (LCP < 2.5s / First Load JS < 200KB)。画像は遅延読み込み。
- **モバイル**: A 案は縦スクロールなので横スクロール起因の崩れは無い。各セクションは縦積みで成立させる。
- **注意 (既知の罠)**: `will-change: transform` は containing block を生む (`reference_will_change_containing_block`)。`position: fixed` 要素との干渉に注意。backdrop-filter の paint コスト (`reference_backdrop_filter_paint_trap`) を LP で多用しない。

### 3c. 原稿と言語

- 今回は **英語 (既定) + 日本語**の 2 言語のみ。
- LP 文言は i18n 翻訳ファイルに **`landing.*` 専用ネームスペース**で新設 (次回の層②で残り 13 言語を足すだけにできる)。既定言語 = 英語、`useI18n()` で出す (層① の仕組みに乗る)。
- 原稿 (見出し・説明文) は Claude が書き、提案 → ユーザー確認の形。ミッション (「整理でなく表現」) に沿わせ、**競合名・戦略ワード・個人情報は出さない** (CLAUDE.md プライバシー原則)。
